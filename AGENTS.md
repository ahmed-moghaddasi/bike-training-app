# Deploy immediately after changes

The rider tests this app on a phone against the deployed build at bike-training-app-gilt.vercel.app, not the local dev server. Local edits are invisible there until pushed. While actively testing a feature together, commit and push to `main` right after making a change (don't batch multiple changes into one push) so the next reload on the phone reflects it.
